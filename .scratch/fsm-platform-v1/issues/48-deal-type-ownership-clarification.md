## Problem

Issue 02 AC#5 (`device.deal_type` manually taggable) was validly deferred — you cannot tag a column
on a table that does not exist yet. But it was deferred **"to Issue 04"**, and `deal_type` appears in
**no** issue between 02 and **35**:

- Issue 04 (snapshot ingestion) and Issue 05 (device state) create `device_states` — neither mentions
  `deal_type` or a commercial-attributes table.
- Issue 35 (Non-Operational dual-confirmation) is the first consumer: a `CONFIRMED` Non-Op marking
  auto-creates a **Recovery Ticket** only when the Device's Deal Type is `RECURRING` (CONTEXT.md
  "Deal Type" / "Non-Operational Marking"; ADR-0014).

So the requirement is currently **orphaned** — at risk of being discovered late, in Issue 35, with no
table column or tagging UI to back it.
